You can override the command properties with msg.payload or msg.command

To override connections settings just sent msg.user, msg.password, msg.host, msg.port

A node that connect with Mikrotik WiFi routers via the RouterOs API.

Configuration

Device: Setup Mikrotik settings like host, port (8278 - default), username and pass.

Action:
  • Log: returns log as msg.payload. Mikrotik command: /log/print
  • Connected WiFi: returns list of registered WiFi devices as msg.payload. Mikrotik command: /interface/wireless/registration-table/print
  • Reboot: reboots Mikrotik. Mikrotik command: /system/reboot
  • Raw: uses msg.payload as command. To pass command with parametrs use JSON object with "command" property as msg.payload. For example: ["/system/script/run","=.id=myscript"]. This example runs script myscript

Output

payload object
The output/result of the executed command on the Mikrotik device.
commandobject
The executed command.
successboolean
true if the command could be successfully executed.

Input

payload / commandobject
This override the executed command.
userstring
This overrides the username.
passwordstring
This overrides the password.
hoststring
This overrides the host.
portnumber
This overrides the port.
